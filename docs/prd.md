# 智能文案平台 - 产品需求文档 (PRD)

## 1. 产品概述
智能文案平台是一款借助大语言模型（LLM）能力，帮助用户通过自然语言对话驱动文案创作、编辑和管理的新一代写作工具。该平台主打“开箱即用”与“AI 强介入”的设计理念，以流畅思考和敏捷创作为核心诉求。

### 1.1 目标用户
- 经常需撰写报告、推文、文章的内容创作者。
- 追求高效排版及灵感激发的办公和营销人员。
- 需要通过配置专属 AI 模型来实现高度定制化文案生成的专业用户。

## 2. 产品结构与核心页面设计
本平台主要分为三个核心界面：**首页（工作空间）**、**创作编辑视图**、**模型配置管理页**。

### 2.1 首页 (Home / Workspace)
**页面布局要求：**
- **极简即时对话框（头部/居中区域）：** 页面核心视觉焦点是一个类似搜索引擎或对话界面的大长条输入框。用户可在此随机输入任何“想法”、“灵感”或“一句话需求”。一旦按下回车（Enter）或发送按钮，系统将自动基于该需求生成文档初稿，并直接跳转进入“编辑页面”。
- **项目管理列表（输入框下方区域）：** 在输入框的下半部分，按时间倒序或自定义分类展示用户所有的历史项目管理及文档页面。此板块支持以卡片或列表的形式呈现历史文稿，方便进行二次浏览和快速进入编辑。

### 2.2 创作编辑页面 (Workspace / Editor)
当用户从首屏输入想法流转进入此页，或点击历史文稿进入此页时，页面采取经典高效的**三栏式布局**：

- **【左栏】大纲视图 (Outline)：**
  提取并展示当前文案的章节、层级结构。支持点击高亮跳转和拖拽重新排序，帮助创作者掌控宏观结构。
  
- **【中栏】文档编辑区 (Document/Editor)：**
  主要的内容呈现与编辑区域。支持富文本编辑及 Markdown 快捷语法；可以支持与 AI 进行基于区块（Block）级别的交互或直接手工修改文本。
  
- **【右栏】AI 智能副驾驶 (AI Copilot / Agent Panel)：**
  借鉴 Cursor 经典的右侧边栏设计。
  - **核心能力：** 提供一个命令行级别的命令输入入口及对话窗，用户可通过对话提出修改意见（例如：“把第二段的语气改得更幽默”、“补充一个关于近期市场的案例”）。
  - **状态与思考流透出：** 在执行用户通过输入框给出的指令时，动态显示底层 AI 操作细节及思考链路（例如：“正在提取文档上下文... -> 正在思考修改逻辑... -> 正在生成文案替代项”）。提升用户的掌控感与 AI 可解析性。

### 2.3 管理与配置页面 (Settings / Management)
用于管理平台偏好及模型配置的独立页面。

- **账户与安全验证：** 页面需通过登录后校验方可访问（支持个人中心管理）。
- **模型绑定与管理中心 (AI Provider Config)：**
  用户可自定义所需的 AI 底层模型，提供默认及自定义接入：
  - **支持 通义千问 (Qwen)**：通过输入对应的 API Key 一键绑定。
  - **支持 OpenAI**：标准的 OpenAI 接口绑定。
  - **兼容 OpenAI 接口的模型站**：提供一个“兼容模式”配置区块，允许填写 `Base_URL` 和 `API_Key`，以灵活支持诸如 DeepSeek、智谱等大量遵守 OpenAI HTTP 标准的 API 端点接入。

## 3. 核心机制 & 使用流程
1. **灵感捕获**：用户打开页面 -> 在首页输入框打字 -> 敲下回车键。
2. **生成并分屏**：后台 AI 根据初始输入生成第一版大纲和正文 -> 自动进入**三栏视图（左大纲、中文档、右 AI）**。
3. **对话式编辑**：用户浏览中栏文稿 -> 发现不满意的内容 -> 在右栏向 AI 提出指令流 -> 右侧呈现 AI 的思考过程和改写计划 -> 中栏文档动态更新变动。
4. **灵活配置**：当平台默认额度耗尽或用户需使用自身模型资源时 -> 进入配置页面绑定 API Key 并即刻生效。
